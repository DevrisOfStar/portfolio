# Portfolio 프로젝트 개선 체크리스트

## 🔴 높은 우선순위 (Critical)

### 보안
- [ ] **CORS 설정 제한**
  - 현재: `Access-Control-Allow-Origin: '*'` (모든 도메인 허용)
  - 개선: 특정 도메인만 허용하도록 제한
  - 파일: `backend/src/routes/portfolio.ts`, `backend/src/routes/blog.ts`
  - 예시: `process.env.ALLOWED_ORIGIN || 'http://localhost:5173'`

- [ ] **이미지 업로드 검증 추가**
  - 현재: 파일 타입/크기 검증 없음
  - 개선: MIME 타입, 파일 크기, 확장자 검증 추가
  - 파일: `backend/src/routes/blog.ts` (이미지 업로드 부분)
  - 검증 항목:
    - 허용된 이미지 타입: `image/jpeg`, `image/png`, `image/gif`, `image/webp`
    - 최대 파일 크기: 5MB
    - 파일 확장자 검증

- [ ] **API 키 관리 개선**
  - 현재: `fetchAdminApi`에 타임아웃 없음
  - 개선: `fetchApi`처럼 타임아웃 추가
  - 파일: `client/src/services/api.js`

### 성능
- [ ] **N+1 쿼리 문제 해결**
  - 현재: `getBlogPosts`에서 각 포스트마다 별도 쿼리 실행
  - 개선: JOIN 또는 배치 쿼리로 최적화
  - 파일: `backend/src/db/queries.ts` (getBlogPosts 함수)
  - 예상 효과: 블로그 포스트가 많을수록 성능 향상

### 입력 검증
- [ ] **날짜 형식 검증**
  - 현재: `body.date` 형식 검증 없음
  - 개선: ISO 8601 또는 YYYY-MM-DD 형식 검증
  - 파일: `backend/src/routes/blog.ts`

- [ ] **ID 검증 추가**
  - 현재: `body.id` 형식/길이 검증 없음
  - 개선: ID 형식 검증 추가 (예: UUID, 특정 패턴)
  - 파일: `backend/src/routes/blog.ts`

- [ ] **이미지 경로 검증**
  - 현재: `path` 파라미터에 `../` 등 경로 조작 가능
  - 개선: 경로 정규화 및 검증
  - 파일: `backend/src/routes/blog.ts` (이미지 업로드 부분)
